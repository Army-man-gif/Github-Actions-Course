Comprehensive Git Command Reference

1. Basic Git Commands

  • git init: Initializes a new Git repository in the current folder. This creates a .git folder to track commits. Local repository starts empty.
  • Connecting to a remote repository manually (after git init):
    • Create remote repo on GitHub.
    • Add it as a remote to your local repo:
      • git remote add origin https://github.com/username/repo.git
    • Push initial commits to GitHub:
      • git push -u origin main
    • This sets origin/main as the tracking branch so future git push and git pull know where to sync.

    This is the alternative to cloning. You manage a local repo first, then connect it to remote manually.

  • git clone <url>: Creates a local copy of a remote repository and automatically sets origin as the tracking remote. Usually used to start work on existing projects.
  • git status: Shows changes in working directory, staged files, and untracked files.
  • git add <file>: Stages files for commit.
  • git commit -m "message": Creates a snapshot of staged changes.
  • git log: Shows commit history. For visual graph:
    • git log --oneline --graph --decorate --all
  • git diff: Shows changes between working directory, staging area, and commits.
  • git checkout <branch/tag/commit>: Switches branch, tag, or commit. If a tag or commit, HEAD becomes detached.

2. Branching

  • git branch <branchname>: Creates a new branch at current commit.
  • git checkout <branchname>: Switch to a branch.
  • git switch <branchname>: Modern branch switch command.
  • git branch -d <branchname>: Deletes a local branch safely.
  • git branch -D <branchname>: Force delete branch.

Branches are moving pointers: they move forward with each commit.

Diverged Branches & Fast-Forward

Diverged branches: your local branch and the remote branch both have commits that the other does not. Example:

Common ancestor B / \ Local D Remote C \ / ?

Git cannot fast-forward in this case. You must either merge or rebase to reconcile.

Fast-Forward: Your branch can be moved directly to the new commit if there are no local changes diverging from the remote:

Remote: A---B---C Local: A---B --fast-forward--> move local pointer to C

3. Tags

  • git tag <tagname>: Creates a lightweight tag.
  • git tag -a <tagname> -m "message": Annotated tag.
  • git push origin <tagname>: Push one tag to remote.
  • git push origin --tags: Push all tags.

Tags are fixed pointers: they never move, unlike branches. Commonly used for marking releases.

4. Merge and Rebase

Merge

  • git merge <branch>: Combines changes from another branch. Creates merge commit if needed.
  • git merge --no-ff <branch>: Forces creation of a merge commit even if fast-forward possible.
A---B---C (main) \ D---E (feature) --merge--> A---B---C------M \ / D---E

Rebase

  • git rebase <base-branch>: Moves local commits on top of base branch, creating linear history.
  • git pull --rebase: Fetches remote changes and replays your commits on top.
Before rebase: A---B---C (main) \ D---E (feature) After rebase on main: A---B---C---D'---E'

5. History Analysis Commands

  • git log: Show commit history.
  • git blame <file>: Shows who last changed each line.
  • git bisect: Binary search for the commit that introduced a bug.
  • git reflog: Shows where HEAD has been, even across resets.

6. Remote Repository Operations

  • git clone <url>: Copy remote repo locally.
  • git fetch: Get remote changes without merging.
  • git pull: Fetch + merge or rebase.
  • git push: Push local commits to tracking remote branch.
  • git push origin <branch/tag>: Push specific branch or tag.
  • git push origin --delete <branch>: Delete remote branch.
  • git remote add <name> <url>: Add new remote.
  • git remote remove <name>: Remove a remote.
  • git remote rename <old> <new>: Rename remote shortcut.

7. Advanced Local Commands

  • git reset --soft HEAD~1: Move HEAD backward but keep staged changes.
  • git reset --hard HEAD~1: Move HEAD backward and discard changes.
  • git stash: Save working directory changes temporarily.
  • git stash pop: Restore stashed changes.
  • git cherry-pick <commit>: Apply a specific commit from another branch.
  • git clean -fd: Remove untracked files and directories.
